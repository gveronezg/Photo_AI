# Portfolio GVG 🚀

Este é um projeto de portfolio pessoal desenvolvido para exibir projetos, habilidades e informações de contato de forma elegante e organizada. O sistema foi construído do zero utilizando o framework Django, focando em boas práticas de desenvolvimento e modularização de código.

## 🛠️ Tecnologias Utilizadas

O projeto foi construído com as tecnologias mais modernas e sólidas do mercado:

-   **Python 3.11+**: Linguagem de back-end robusta e eficiente.
-   **Django (5.x)**: Framework web "com baterias incluídas" para garantir segurança e escalabilidade.
-   **HTML5 & CSS3**: Estrutura e estilização responsiva com design moderno e sombrio (Dark Mode).
-   **Google Fonts (Inter)**: Tipografia otimizada para leitura e estética premium.

## 🧠 Conceitos Trabalhados

Durante o desenvolvimento, foram aplicados conceitos fundamentais de engenharia de software e desenvolvimento web:

-   **Herança de Templates (DRY - Don't Repeat Yourself)**: Uso do arquivo `base.html` para centralizar a estrutura do site, permitindo que as páginas reutilizem o cabeçalho e rodapé.
-   **Modularização de CSS Profissional**: Divisão do código CSS em arquivos específicos (`base`, `components`, `pages`), melhorando a performance e facilite a manutenção do projeto.
-   **Mapeamento Dinâmico de Rotas**: Configuração de URLs amigáveis com passagem de parâmetros (ex: `/projetos/<id_projeto>/`).
-   **Gerenciamento de Contexto no Django**: Passagem e exibição de dados dinâmicos (`dicionários Python`) diretamente nas views para os templates.
-   **Arquivos Estáticos (Static Files)**: Organização e configuração correta do Django para servir CSS e outros recursos do servidor.

## 🌟 Experiências e Aprendizados Obtidos

Realizar este projeto trouxe evoluções significativas em diversas áreas:

1.  **Estrutura de Projetos Reais**: Aprendizado total sobre como iniciar e organizar um projeto Django seguindo fluxos de trabalho profissionais.
2.  **Clean Code e Organização**: Prática de manter o código separado e bem identificado, facilitando futuras manutenções e escalabilidade.
3.  **Resolução de Problemas (Debug)**: Experiência prática na resolução de erros comuns de caminhos de arquivos e tags de templates (como o `{% load static %}`).
4.  **Modularização do CSS**: Entendimento profundo sobre como separar estilos globais de estilos específicos de página para otimizar o carregamento.
